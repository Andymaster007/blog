Opening this blog with a quick note on its purpose, so I don't forget later.

My personal homepage (`litmustz.pages.dev`) has a "Projects" section that holds **a one-page overview of each project once it is done** — background, tech stack, architecture, challenges, results. That is for someone learning about the project for the first time.

This blog has a different purpose. It does not hold project overviews; it holds **the "other stuff" that comes up while building things**:

- How I solved a specific tricky problem (say, frame-rate-independent movement);
- Dev logs and the pitfalls I hit along the way;
- A point that suddenly clicked halfway through reading something;
- Some scattered technical retrospectives.

Take my current project **SDLSpaceShooter** (a 2D space shooter written from scratch in C++17 + SDL2) as an example: its overview lives on the project page of my homepage, while the concrete questions I ran into mid-development — "how to handle multi-resolution scaling", "which directory should saves actually write to" — get written here.

So this blog is more like my **thinking scratchpad** — not always complete, but all real things that came up while I was actually doing the work. If any post is useful to you too, that is the best outcome.

I will keep filling it in slowly, starting with the noteworthy bits from the projects I am working on right now.
