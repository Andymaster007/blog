# 原始素材库（Source Archive）

> **这是整个博客的「第一层 / 唯一真相来源」。** 展示层（网站）如何改版、换主题、换框架，
> 只要这个库还在，我就能一键重新生成整站，零丢失。

## 为什么要有这一层

博客的「内容」和「展示」是两回事，必须分开存：

- **`_source-archive/`**（本目录）= 你发给我的**原始素材**：原话、原图、原始数据。
  它**永远不被 Hexo 处理**，只作存档与再生成之用，纳入 Git 版本管理。
- **`source/_posts/`** = 我加工后的**展示层文章**（Markdown + 资源文件夹），由 Hexo 构建成网页。

改版时我只动展示层，原始素材纹丝不动；想换版面/换主题时，从素材库重新生成即可。

## 单条内容的目录结构

每条内容一个文件夹，命名用「日期-简短slug」，例如 `2026-07-18-heap-sort-notes/`：

```
_source-archive/
└── 2026-07-18-heap-sort-notes/
    ├── raw.md        # 你传给我的原话，一字不改
    ├── meta.yml      # 结构化元信息（见下）
    └── assets/        # 你发的原图 / 原文件，原分辨率不压缩
        └── diagram.png
```

### meta.yml 字段约定

```yaml
title: 堆排序学习笔记              # 文章标题
date: 2026-07-18                  # 发布日期
category: 算法心得                # 主板块（见下方「板块清单」）
tags: [排序, 数据结构]            # 标签数组
project: ''                      # 所属项目名（项目类文章填写）
type: essay                      # essay=随笔 / project=案例式项目复盘
excerpt: ''                      # 首页摘要（留空我帮你写）
cover: ''                        # 封面图路径（留空用默认）
note: ''                         # 我加工时的备注（可选）
```

## 板块清单（categories，导航栏分区用）

1. **算法心得** —— 刷题 / 数据结构 / 算法思路
2. **游戏项目** —— 游戏开发相关项目复盘
3. **金融项目** —— 金融相关项目与实践
4. **编程技术杂谈** —— 日常写代码心得、踩坑、工具

> 文章默认按日期倒序展示（最新的在最上面），板块通过导航栏「分类」进入。

## 代运营工作流（你口述 → 上线）

1. 你发我文字 / 图片 / 心得。
2. 我先原样存进 `_source-archive/<id>/`（raw.md + assets + meta.yml）。
3. 我基于素材加工出 `source/_posts/<slug>.md` 及资源文件夹。
4. `hexo generate` 构建，本地 `hexo server` 预览给你确认。
5. 你点头 → 我 push 到 GitHub → Cloudflare Pages 自动上线。

## 备份策略（三重保险）

- 本机文件夹（`_source-archive/` 与 `source/` 都在仓库内）
- Git 完整版本历史（每次改动可回溯）
- GitHub 异地云端（Cloudflare 自动部署的来源）
