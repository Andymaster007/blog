## 00:51 修复：博客切换后滚动位置不恢复 + 中文侧脚本漏改
- 现象：点切换能留在同篇，但滚动位置不恢复（回顶部）。
- 根因①：restoreScroll 只在 load/300ms 各 scrollTo 一次，被 Fluid 主题 load 后回顶/淡入或图片撑高覆盖。
- 根因②（隐藏坑）：上一轮只对 `source-en/js/lang-switch.js` 改了新版 restoreScroll，`source/js/lang-switch.js` 的 Edit 实际未落盘，中文侧一直是旧逻辑。
- 修复：两文件统一改为「关闭 history.scrollRestoration + setInterval 每 50ms 重试压回目标位置，持续约 1.8s，直到到位或用户自己滚动/触屏/按键即停手」。
- 验证：public/blog 与 public/en/blog 的 lang-switch.js 均含 setInterval（计数=1），4900 端口预览返回新版。
- 教训：改对称双文件（source/ 与 source-en/）时必须逐文件确认落盘，不能假设一次 Edit 对两侧都生效；改动后务必 grep 验证两侧产物。
